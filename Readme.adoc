== Über dieses Projekt

Dieses Repository enthält einen Implementierungsleitfaden für Hersteller von Apps für Versicherte.
Da die App sich aus verschiedenen Modulen zusammensetzt, die jeweils mit eigenen Backend-Services kommunizieren, ist die Dokumentation in mehrere Teile gegliedert:

== Inhalte

=== link:docs/app_overview.adoc[App-Basis]

Beschreibt die grundlegende Architektur der App und Basisanwendungsfälle, wie das initiale Anlegen einer GesundheitsID oder die von den Zeta Clients verwendeten Parameter des Devices.

=== link:modules/DiPag/docs/AF_Overview.adoc[Übersicht der DiPag-Anwendungsfälle]

Listet alle fachlichen Anwendungsfälle der DiPag auf, gegliedert nach den Bereichen Übermittlung, Empfang und Verwaltung von Rechnungen, Einreichung beim Kostenträger, Einrichtung von Nutzerkonten sowie Nutzerprotokolle. Jeder Anwendungsfall ist einzeln als AsciiDoc-Datei dokumentiert und aus der Übersicht heraus verlinkt.
